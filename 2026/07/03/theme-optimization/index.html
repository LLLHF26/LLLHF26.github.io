博客折腾记：给 NexT 加音乐播放器、换字体和性能优化

发表于 2026-07-03 更新于 2026-07-07 分类于 日常碎碎念

折腾了一整天，终于把博客调成了自己想要的样子。这篇简单记录一下这次对 NexT 主题做的几处优化，也算给以后再踩坑的自己留个备忘。

一、给侧边栏加网易云音乐播放器

用的是 APlayer + MetingJS 的组合，在侧边栏「站点概览」里嵌一个歌单播放器。核心就是在 _layout.swig 引入两个库，再在 sidebar.swig 里放一个 <meting-js> 标签指向自己的网易云歌单。

看着简单，实际踩了一路坑：

  • 广告拦截插件：一开始播放器怎么都不出来，控制台一堆 ERR_BLOCKED_BY_CLIENT，最后发现是浏览器的广告拦截插件把 CDN 请求整个掐了，白名单放行就好。
  • CDN 被墙：jsdelivr 在国内加载不稳定，换成了 staticfile.org 国内源。
  • 歌单 ID 无效：一定要用公开歌单，私有歌单接口直接报 unknown playlist id。
  • 神秘白痕：播放器中间有几条白线，查了半天发现是 APlayer 歌词区的白色渐变遮罩，在深色背景下显成了白条，直接把歌词区隐藏掉解决。
  • 版权限制：热门 / VIP 歌曲拿不到音频源，点播放会报 no supported source。这个没法绕过，只能在歌单里放能免费播放的歌（纯音乐、古典、无版权的基本都行）。

二、完善侧边栏个人信息

把默认的「John Doe + 一堆计数」换成了真正的个人区：头像、昵称、一句签名、以及邮箱社交链接。头像记得放进 source/images/，因为 Hexo 只发布 source 目录里的东西。

三、全站换字体：霞鹜文楷 LXGW WenKai

一直很喜欢霞鹜文楷那种手写楷体的书卷感，这次直接给全站换上了。用的是按需子集加载的 webfont，不会一次性下载几十 MB。同时保留了代码块的等宽字体、也没动图标字体。

四、去掉页脚的「Powered by」

主题配置里把 footer.powered 设为 false，页脚那行「由 Hexo 强力驱动」就没了，清爽。

五、性能优化：解决跳转卡顿

这个是重点。之前每次点链接跳转都要卡好久白屏，排查下来是两个原因叠加：

  1. 字体 CSS 渲染阻塞——浏览器要等 CDN 把字体 CSS 下载完才肯渲染页面。改成 media="print" onload 的非阻塞加载后，页面秒出，字体好了再无感切换。
  2. 动画等资源——NexT 的 motion 动画默认等整页资源加载完才开始，内容一直”隐身”。把 motion.async 改成 true，动画立即开始，内容马上就出来了。

改完之后跳转明显顺滑了很多。

写在最后

一顿折腾下来，博客总算有了点自己的味道。建站这件事，麻烦是真麻烦，但把每个小细节调到顺眼的那一刻，还挺爽的。下次有新坑再来更新。
