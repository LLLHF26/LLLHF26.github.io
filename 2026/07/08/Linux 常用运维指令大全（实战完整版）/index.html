Linux 常用运维指令大全

本文整理运维工作中高频使用、生产必备的Linux指令，按场景分类，涵盖文件目录、权限管理、磁盘资源、进程管理、网络运维、日志查看、软件包管理、系统运维、文件传输、批量运维十大核心场景，所有指令均附带常用参数、实操示例及适用场景，适配CentOS、Ubuntu主流服务器系统，可直接用于线上运维排查、日常维护。

一、文件与目录操作（最基础高频）

主要用于服务器文件浏览、创建、移动、删除、查找等日常操作，是所有运维操作的基础。

指令 核心功能 常用参数+实操示例 运维场景
ls 列出目录文件/属性 ls -l（详细信息）、ls -a（显示隐藏文件）、ls -lh（人性化显示大小）、ls -lt（按修改时间排序） 查看目录文件、排查文件大小、核对文件修改时间
pwd 查看当前工作目录绝对路径 直接输入pwd执行，无额外参数 定位当前操作目录，避免路径错误
cd 切换工作目录 cd /home（进入指定目录）、cd ..（返回上一级）、cd ~（回到用户家目录）、cd -（返回上一次目录） 目录跳转、切换工作路径
mkdir 创建目录 mkdir test（创建单目录）、mkdir -p /a/b/c（递归创建多级目录） 搭建项目目录、创建日志存储目录
rm 删除文件/目录 rm 文件名（删文件）、rm -rf 目录名（强制递归删除目录，无提示） 清理垃圾文件、删除无用目录（生产慎用-rf）
cp 复制文件/目录 cp file1 file2（复制文件）、cp -r 目录1 目录2（复制目录）、cp -p（保留文件属性） 备份配置文件、迁移项目文件
mv 移动/重命名文件 mv 旧名 新名（重命名）、mv 文件 /目标路径（移动文件） 文件归类、配置文件替换、项目迁移
find 全局查找文件/目录 find / -name “*.log”（全盘查找日志文件）、find ./ -size +100M（查找当前目录大于100M的文件） 排查大文件、查找缺失配置文件、清理过期日志

二、文件查看与编辑指令

用于查看服务器日志、编辑配置文件，是故障排查、服务配置的核心指令。

指令 核心功能 常用参数+实操示例 运维场景
cat 查看小文件全部内容 cat nginx.conf（查看配置文件）、cat file1 >> file2（文件内容合并） 查看小配置文件、快速核对文件内容
less 分页查看大文件（可上下翻页） less 日志文件，空格翻页、q退出、/关键词搜索 查看超大日志文件，避免cat刷屏卡顿
head 查看文件头部内容 head -n 20 日志文件（查看前20行） 快速查看日志开头、配置文件头部参数
tail 查看文件尾部内容（日志必备） tail -n 30 日志（查看最后30行）、tail -f 日志（实时监控日志刷新） 线上故障实时排查、监控服务运行日志
vi/vim 文件编辑（系统默认编辑器） vim 文件名，i进入编辑、ESC退出编辑、:wq保存退出、:q!强制不保存退出 修改服务配置文件、编写脚本、配置环境参数
grep 文件内容关键词过滤 grep “error” 日志文件（筛选报错日志）、grep -v “空行”（过滤指定内容）、grep -r “关键词” ./（递归筛选目录下所有文件） 排查日志报错、筛选配置参数、检索关键信息

三、权限与用户管理指令

用于服务器账号管理、文件权限管控，保障服务器访问安全，解决权限不足、访问报错等问题。

指令 核心功能 常用参数+实操示例 运维场景
chmod 修改文件/目录权限 chmod 755 文件名（读写执行权限）、chmod -R 775 目录（递归修改目录所有文件权限） 解决文件权限不足、服务无法读取文件问题
chown 修改文件/目录所属用户和组 chown root:root 文件名、chown -R nginx:nginx 项目目录 服务权限归属配置、解决文件属主错误导致的访问异常
useradd 创建系统用户 useradd test（创建test用户）、useradd -m test（创建用户并生成家目录） 新增运维账号、业务专属用户
userdel 删除系统用户 userdel test（删除用户）、userdel -r test（删除用户及家目录） 清理废弃账号、注销离职人员运维权限
passwd 修改用户密码 passwd（修改当前用户密码）、passwd test（修改指定用户密码） 账号密码重置、定期更新服务器密码
sudo 临时获取管理员权限 sudo 指令（普通用户执行管理员命令）、sudo -i（切换root权限） 普通用户运维操作、临时授权执行高危指令

四、磁盘与资源监控指令

监控服务器磁盘、内存、空间资源，排查磁盘爆满、内存溢出、资源占用过高等问题。

指令 核心功能 常用参数+实操示例 运维场景
df 查看磁盘分区空间占用 df -h（人性化显示磁盘剩余/占用空间）、df -i（查看inode使用情况） 排查磁盘爆满、分区空间不足、inode耗尽问题
du 统计文件/目录磁盘占用大小 du -sh 目录（查看目录总大小）、du -h --max-depth=1（查看当前目录各级文件大小） 定位大文件、清理占用空间过高的目录
free 查看内存/交换分区使用情况 free -h（人性化显示内存、缓存、剩余内存） 排查内存溢出、内存占用过高、服务卡顿问题
mount/umount 挂载/卸载磁盘分区 mount /dev/sdb1 /data（挂载磁盘）、umount /data（卸载挂载目录） 服务器新增磁盘、挂载数据盘、迁移数据分区

五、进程与服务管理指令

管控服务器运行进程、系统服务，排查进程卡死、服务宕机、端口占用等核心问题，是运维故障排查核心。

指令 核心功能 常用参数+实操示例 运维场景
ps 查看系统进程快照 ps -ef（查看所有进程）、ps -ef | grep nginx（筛选指定服务进程） 核对服务是否启动、排查异常进程
top/htop 实时监控进程资源占用 top（系统自带，实时查看CPU/内存占用）、htop（彩色可视化监控，需yum/apt安装） 排查服务器CPU爆满、内存飙升、进程卡死问题
kill 终止进程 kill 进程PID（正常终止）、kill -9 进程PID（强制杀死卡死进程） 关闭异常进程、重启卡死服务
systemctl 系统服务管理（CentOS7+/Ubuntu16+） systemctl start nginx（启动服务）、stop（停止）、restart（重启）、status（查看状态）、enable（开机自启）、disable（关闭自启） 管理Nginx、MySQL、Java等所有系统服务
netstat/ss 查看端口、连接状态 netstat -tulpn（查看所有监听端口）、ss -tulpn（高效端口查询，替代netstat） 排查端口占用、端口冲突、服务监听异常

六、网络运维指令

排查服务器网络连通性、IP配置、域名解析、外网访问等网络问题，解决ping不通、端口不通、网络延迟等故障。

指令 核心功能 常用参数+实操示例 运维场景
ip 查看/配置网络信息（替代ifconfig） ip addr（查看本机IP、网卡信息）、ip route（查看路由表） 核对服务器IP、网卡状态、路由配置
ping 测试网络连通性 ping 域名/IP、ping -c 4 地址（仅发送4次数据包，停止刷屏） 排查服务器外网连通、跨服务器通信异常
curl/wget 网络请求/文件下载 curl 网址（测试接口访问）、wget 链接（下载网络文件） 排查接口不通、下载安装包、测试外网访问
traceroute 追踪网络路由节点 traceroute 目标IP/域名 排查网络延迟、丢包、路由跳转异常
nslookup/dig 域名DNS解析查询 nslookup 域名、dig 域名（详细解析信息） 排查域名解析失败、DNS劫持、解析延迟问题

七、软件包管理指令

适配CentOS（yum）、Ubuntu（apt）两大系统，用于软件安装、升级、卸载、依赖修复。

1、CentOS/RHEL 系统（yum）

  • yum install 软件名：安装软件包（例：yum install nginx -y）

  • yum remove 软件名：卸载软件（保留配置文件）

  • yum erase 软件名：彻底卸载软件（删除配置）

  • yum update：升级系统所有可更新软件包

  • yum list | grep 软件名：查询软件是否可安装

  • yum clean all：清理yum缓存，释放磁盘空间

2、Ubuntu/Debian 系统（apt）

  • apt update：更新软件源缓存

  • apt install 软件名：安装软件包

  • apt remove 软件名：卸载软件（保留配置）

  • apt purge 软件名：彻底卸载+清理配置

  • apt upgrade：升级已安装软件包

  • apt full-upgrade：完整升级（自动清理冗余依赖）

八、文件传输与压缩解压指令

用于服务器文件上传、下载、压缩打包、解压，是文件迁移、备份的必备指令。

1、压缩解压（tar 万能指令）

  • 打包压缩：tar -zcvf 压缩包名.tar.gz 目标目录/文件

  • 解压文件：tar -zxvf 压缩包名.tar.gz

  • 解压到指定目录：tar -zxvf 压缩包名.tar.gz -C /目标路径

  • 参数释义：z（gzip压缩）、c（创建压缩包）、x（解压）、v（显示过程）、f（指定文件）

2、文件传输

  • scp：跨服务器文件传输
    示例：scp 本地文件 远程用户@远程IP:/远程路径（本地上传远程）
    示例：scp 远程用户@远程IP:/远程文件 本地路径（远程下载本地）

  • rz/sz：本地与服务器互传（需安装lrzsz）
    rz：本地文件上传服务器；sz 文件名：服务器文件下载到本地

九、系统运维基础指令

用于系统重启、关机、时间查看、版本查询、负载监控等基础系统操作。

  • uname -a：查看系统内核、版本信息

  • cat /etc/os-release：查看服务器系统版本（CentOS/Ubuntu）

  • hostname：查看/修改服务器主机名

  • uptime：查看服务器运行时长、系统负载

  • date：查看/修改系统时间

  • shutdown -h now：立即关机

  • shutdown -r now / reboot：立即重启服务器

  • history：查看历史执行指令

十、运维高频组合指令（实战神器）

日常故障排查、批量操作高频组合指令，大幅提升运维效率

  • 筛选日志报错：grep -i "error|warn" app.log（忽略大小写筛选错误、警告日志）

  • 实时监控报错日志：tail -f app.log | grep "error"

  • 查找100M以上大文件：find / -type f -size +100M | sort -rh

  • 查看端口占用进程：netstat -tulpn | grep 端口号

  • 批量删除指定后缀文件：find ./ -name "*.tmp" -delete

  • 统计日志访问量：grep "200" access.log | wc -l

十一、运维操作注意事项

  1. 慎用 rm -rf：禁止直接执行 rm -rf / 根目录删除，删除前务必核对路径

  2. 高危操作先备份：修改配置、删除文件前，先备份原文件（cp 原文件 原文件.bak）

  3. 禁止随意kill进程：生产环境优先用kill正常终止，kill -9仅用于卡死无响应进程

  4. 权限最小化原则：非必要不设置777权限，避免服务器安全风险

  5. 服务重启先检查：重启服务前先查看status状态，避免配置错误导致服务启动失败
