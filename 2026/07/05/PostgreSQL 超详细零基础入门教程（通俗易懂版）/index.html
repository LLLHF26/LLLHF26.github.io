PostgreSQL 超详细零基础入门教程

前言

PostgreSQL（简称 PG）是一款开源、免费、高性能、高兼容、支持拓展性极强的关系型数据库，相比 MySQL，它支持更丰富的数据类型、事务完全合规、自带分区、索引、JSON 高级处理能力，是企业级后端、大数据、物联网项目的主流数据库之一。

本教程全程零基础友好，从安装、基础命令、SQL 语法、表操作、数据增删改查、进阶特性、实战优化一步步讲解，所有代码可直接复制运行。

一、PostgreSQL 核心特点（必懂）

  • 完全开源免费：无商业版权风险，个人、企业均可免费使用

  • 事务强一致性：完整支持 ACID 事务，金融、支付等对数据安全要求高的场景首选

  • 数据类型丰富：支持常规数值、字符串，还支持 JSON/JSONB、数组、地理坐标、时间戳、二进制等特殊类型

  • 拓展性极强：支持自定义函数、存储过程、触发器、自定义数据类型

  • 高性能稳定：支持并行查询、分区表、读写分离，大数据量场景表现优异

  • 标准 SQL 兼容：兼容绝大多数 ANSI SQL 标准，学习成本低

二、环境安装与配置（Windows/Mac/Linux）

2.1 Windows 安装

  1. 官网下载：进入 PostgreSQL 官网下载对应系统版本的安装包

  2. 双击安装，默认下一步即可，关键配置：

    • 安装路径：建议纯英文路径，无中文、无空格

    • 设置超级管理员密码（务必记住，默认用户名：postgres）

    • 默认端口：5432（无需修改）

  3. 安装完成后，勾选启动 Stack Builder（可选，用于安装插件），也可直接跳过

  4. 打开自带客户端 pgAdmin4，即可可视化操作数据库

四、PostgreSQL 常用数据类型

建表必须指定字段类型，整理高频实用类型，适配所有业务场景。

4.1 数值类型

  • smallint：短整型（-32768 ~ 32767）

  • int / integer：整型（常用，默认）

  • bigint：长整型（主键、自增ID首选）

  • numeric(m,n)：高精度小数（金额、财务数据专用，m总位数，n小数位数）

  • float/double：浮点型（非精准计算使用）

4.2 字符串类型

  • char(n)：固定长度字符串，不足补空格

  • varchar(n)：可变长度字符串（姓名、手机号常用）

  • text：无限长字符串（文章、详情描述专用，无长度限制）

4.3 时间日期类型

  • date：日期（仅年月日：2026-07-07）

  • time：时间（仅时分秒：12:00:00）

  • timestamp：时间戳（年月日+时分秒，最常用）

  • timestamptz：带时区时间戳（跨时区项目首选）

4.4 PG 特色类型（核心优势）

  • boolean：布尔类型（true/false）

  • JSON / JSONB：JSON类型（存储前端参数、拓展字段，JSONB支持索引，查询更快）

  • 数组类型：可直接存储数组，如 int[]、varchar[]

十一、常见报错与解决方案

  1. 密码错误：重新修改超级用户密码，ALTER USER postgres WITH PASSWORD ‘新密码’;

  2. 端口5432被占用：修改 PG 配置文件端口，或关闭占用进程

  3. 外键关联报错：删除数据时需先删除关联子表数据

  4. JSON格式错误：插入 JSON 数据需严格符合 JSON 格式，引号配对

十二、学习总结与进阶方向

12.1 入门核心重点

熟练掌握：数据库/用户管理、表结构设计、CRUD 语法、联查、聚合分组、JSONB 操作、索引基础，可满足日常开发 90% 需求。

12.2 进阶学习方向

  • 高级特性：分区表、触发器、存储过程、自定义函数

  • 性能优化：慢查询分析、执行计划、读写分离、主从复制

  • 实战场景：海量数据存储、日志存储、地理位置数据处理
